# 文章修订历史 diff 功能 — 实施计划

## 目标
在每篇文章详情页末尾，自动展示该文的**全部历史版本**与**当前最新版**的**行级 diff**（服务端渲染，构建时生成 HTML，前端零 JS 依赖）。

## 设计决策（已与你确认）
- **渲染方式**：服务端渲染——构建时（`hexo generate`）在 Node 脚本里算好 diff HTML，直接写进文章页。前端零 JS、首屏即见。
- **历史范围**：全部历史版本——列出该文件的所有 git 提交，每个版本可点开展示「该版本 → 当前最新版」的行级 diff。
- **diff 对象**：对 markdown 源码做行级 diff（不是渲染后的 HTML），diff 干净直观。
- **展示形式**：`<details>` 折叠的修订时间线，默认折叠，点开看具体 diff。

## 技术架构

利用 Hexo 三层机制，**不改主题源码**：

```
scripts/git-revision.js          ← before_generate 钩子：读 git 历史 + 算 diff + 挂到 post 对象
       │
       ▼
post.full_source (绝对路径)       ← 遍历所有文章
       │
       ▼ git log --follow + git show <hash>:path
       │
       ▼ diff 库算行级 diff → 生成 HTML 字符串
       │
       ▼ post.revisionHtml = "<details>...</details>"
       │
       ▼ template_locals filter 把 revisionHtml 暴露到 page.revisionHtml
       │
source/_data/post-body-end.njk   ← NexT postBodyEnd 注入点：{{ page.revisionHtml | safe }}
       │
       ▼ _config.next.yml 的 custom_file_path.postBodyEnd 注册
       │
文章页 HTML 末尾                  ← 静态写入，部署即可见
```

## 新增依赖

**`diff`**（npm 包，~40KB，纯 JS 无原生编译，29M 周下载的事实标准 diff 库）
- 用途：`diff.createPatch` / `diffLines` 算行级 diff
- 为什么不用纯 git 解析：自己解析 unified diff 跨平台/边界情况易错
- 不违反 AGENTS.md 红线 2（该红线是"不要删现有依赖"，加新功能必需依赖是合理的）

### 2. `scripts/git-revision.js`（新建，自动加载）

核心逻辑（伪代码）：
```js
/* global hexo */
const { execSync } = require('child_process');
const path = require('path');
const Diff = require('diff');

// before_generate: 跑一次，遍历所有文章算 diff
hexo.extend.filter.register('before_generate', () => {
  const posts = hexo.locals.get('posts');
  const baseDir = hexo.base_dir;

  posts.each(post => {
    try {
      const relPath = path.relative(baseDir, post.full_source).replace(/\\/g, '/');
      // 1. 拿该文件所有提交: git log --follow --pretty=format:%H|%ci|%s -- <path>
      const log = execSync(`git log --follow --pretty=format:%H|%ci|%s -- "${relPath}"`, {
        cwd: baseDir, encoding: 'utf8'
      }).trim();

      if (!log) return;  // 无历史（新文件未提交）静默跳过

      const commits = log.split('\n').map(line => {
        const [hash, date, ...msgParts] = line.split('|');
        return { hash, date, message: msgParts.join('|') };
      });

      // 2. 对每个历史版本，git show <hash>:<path> 拿旧内容，与当前内容算 diff
      const currentContent = post.raw;  // 当前 markdown 源码
      const revisions = commits.map(c => {
        const oldContent = execSync(`git show ${c.hash}:"${relPath}"`, {
          cwd: baseDir, encoding: 'utf8'
        });
        const diff = Diff.diffLines(oldContent, currentContent);
        return { ...c, diff };
      });

      // 3. 生成 HTML（<details> 折叠时间线）
      post.revisionHtml = renderRevisions(revisions);
    } catch (e) {
      // git 不可用/无历史等：静默跳过，不阻断构建
      hexo.log.warn(`git-revision: 跳过 ${post.path} - ${e.message}`);
    }
  });
}, 10);  // 优先级 10，确保在 NexT 的 before_generate(0) 之后跑

// template_locals: 把 revisionHtml 暴露到模板
hexo.extend.filter.register('template_locals', locals => {
  // post 对象上的 revisionHtml 已自动作为 page.revisionHtml 可见，此处无需额外处理
  // 但为保险显式传递
  return locals;
});

function renderRevisions(revisions) {
  if (revisions.length <= 1) return '';  // 只有1个提交=无历史改动，不显示
  // 生成 <details><summary>日期 + 提交说明</summary><div class="revision-diff">...</div></details>
  // diff 块: <del>删除行</del> / <ins>新增行</ins> / <span>不变行</span>
  // ...
}
```

**降级保证**：任何 git 错误都 try-catch 静默跳过，只 `hexo.log.warn`，绝不阻断构建（符合 AGENTS.md"CI 不能因新功能挂掉"的隐含要求）。

**只对详情页生效**：postBodyEnd 注入点在 NexT 的 `post.njk` 里只在非首页（`!is_index`）时渲染，天然不污染首页摘要列表。

## diff 展示形态（最终页面效果）

文章正文末尾出现：

```
修订历史
─────────────
▸ 2026-07-06 post: 长风破浪          ← 初始发布（折叠）
▸ 2026-07-08 update: 改了结尾两句    ← 点开展示行级 diff
    - 行路难，行路难，多歧路，今安在？（旧）
    + 行路难，多歧路，今安在？（新）
▸ 2026-07-09 update: 加一段          ← 当前版本（最新）
```

- `<details>` 默认全折叠，点 `▸` 展开
- 删除行 `<del>` 红色背景，新增行 `<ins>` 绿色背景（内联 `<style>` 在 njk 文件里，避免额外 styl 文件）
- 最新版本（HEAD）不显示 diff（它就是当前正文），只标注"当前版本"
- 只有 1 个提交的文章不显示修订历史区块（没有改动可对比）

## 风险与缓解

| 风险 | 缓解 |
|------|------|
| CI fetch-depth:0 拉全量历史变慢 | 本仓库小（~20 提交），增量可控 |
| git 命令在 Windows 本地路径分隔符问题 | `.replace(/\\/g, '/')` 统一正斜杠 |
| 中文 commit message 编码 | execSync 设 `encoding: 'utf8'` |
| 某文章无 git 历史（新建未提交） | try-catch 静默跳过 |
| diff 库体积 | ~40KB，仅构建期用，不进前端 bundle |
| 违反"纯记录型"定位 | 功能默认折叠、不打扰正文，纯属可选查看 |

## 不做的事
- 不改已发布文章的 permalink
- 不改主题源码（只用 custom_file_path 注入）
- 不加前端 JS 库（纯服务端渲染）
- 不动 `_config.yml` 的 url/root
- 不装 hexo-renderer-nunjucks
